# HUDS GUI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build three independent GUI implementations (PyQt5 Wizard, PyQt5 Tabs, PySide6 Modern) plus a unified launcher for the HUDS active learning workflow.

**Architecture:** Three isolated GUI folders each import `huds_app.*` core layer. Each has its own worker thread for background simulation/training. `launch_gui.py` provides a menu to select which GUI to run.

**Tech Stack:** PyQt5, PySide6, pyqtgraph, QtMaterial, QThread

- [ ] **Step 2: Create worker.py — Background thread for HUDS workflow**

File: `gui_wizard/worker.py`

Contains a `QThread` subclass that runs the full HUDS loop (config → AEDT connect → simulation → training). Exposes signals:
- `progress_signal(int)` — overall progress percentage
- `log_signal(str)` — log messages to append
- `r2_signal(float)` — R2 value update for chart
- `step_signal(str)` — current step name (e.g., "Sampling", "Simulating", "Training")
- `finished_signal(bool, str)` — success/failure + message

The worker receives a config dict and run_dir path as constructor args. It calls the same functions as `launch_huds.py` but emits signals instead of printing.

- [ ] **Step 3: Create config_page.py — Configuration form**

File: `gui_wizard/pages/config_page.py`

QWizardPage subclass with form fields:
- QLineEdit for AEDT project path (with browse button)
- QLineEdit for variable names, mins, maxs, units (comma separated)
- QLineEdit for output names (comma separated)
- QDoubleSpinBox for steady_state_pct (default 0.2, range 0.01-1.0)
- QSpinBox for total_samples (default 600), initial_train (default 20), sample_per_step (default 15), max_steps (default 3), epochs (default 200)
- QComboBox for device (cpu/cuda)
- `validatePage()` checks field consistency, stores config dict in wizard property

- [ ] **Step 4: Create aedt_page.py — AEDT connection and project selection**

File: `gui_wizard/pages/aedt_page.py`

QWizardPage subclass with:
- A "Connect" button that calls `connect_aedt_auto()` logic from worker
- QListWidget showing available projects (populated after connection)
- Project index selector or file path input for new project
- Design list and selector after project is opened
- `validatePage()` ensures project and design are selected

- [ ] **Step 5: Create monitor_page.py — Simulation and HUDS loop monitoring**

File: `gui_wizard/pages/monitor_page.py`

QWizardPage subclass with:
- QProgressBar for overall progress
- QTextBrowser for live log output (connected to worker.log_signal)
- QLabel showing current step name
- "Start" button that launches the worker thread
- "Abort" button that signals worker to stop
- Page is read-only until user clicks Start

- [ ] **Step 6: Create result_page.py — Results visualization**

File: `gui_wizard/pages/result_page.py`

QWizardPage subclass with:
- pyqtgraph PlotWidget showing R2 curve over steps
- QTableWidget showing metrics per step (R2, RMSE)
- "Export CSV" button to save results
- Populated from worker.r2_signal history and final state.json

- [ ] **Step 7: Create wizard.py — Main wizard orchestrator**

File: `gui_wizard/wizard.py`

QWizard subclass that adds config_page, aedt_page, monitor_page, result_page in order. Connects wizard properties between pages (config dict flows from config_page to worker via aedt_page).

### Task 4: launch_gui.py — Unified Launcher

**Files:**
- Create: `launch_gui.py` (project root)

**Interfaces:**
- Consumes: gui_wizard.main, gui_tabs.main, gui_modern.main (via subprocess or import)
- Produces: CLI menu launcher

- [ ] **Step 1: Create launch_gui.py**

```python
import os
import sys

PROJECT_ROOT = os.path.dirname(os.path.abspath(__file__))
sys.path.insert(0, PROJECT_ROOT)
os.environ["ANSYSLMD_LICENSE_FILE"] = "24500@licensing.hkust.edu.cn"

PYTHON = r"D:\miniconda\envs\agents\python.exe"

def main():
    print("=" * 40)
    print("HUDS GUI Launcher")
    print("=" * 40)
    print("[1] Wizard  (PyQt5 step-by-step)")
    print("[2] Tabs    (PyQt5 multi-panel)")
    print("[3] Modern  (PySide6 + Material)")
    
    choice = input("Choice [1]: ").strip() or "1"
    
    options = {
        "1": os.path.join(PROJECT_ROOT, "gui_wizard", "main.py"),
        "2": os.path.join(PROJECT_ROOT, "gui_tabs", "main.py"),
        "3": os.path.join(PROJECT_ROOT, "gui_modern", "main.py"),
    }
    
    target = options.get(choice)
    if not target:
        print("Invalid choice")
        sys.exit(1)
    
    import subprocess
    subprocess.run([PYTHON, target])

if __name__ == "__main__":
    main()
```

- [ ] **Step 2: Test launcher**

Run: `D:\miniconda\envs\agents\python.exe launch_gui.py`
Expected: Menu displays, selecting 1/2/3 launches corresponding GUI
